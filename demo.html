Cordova Research

Apache Cordova allows you to write HTML, CSS & Javascript files that can be hosted in an application container.

I investigated a hosted app solution, which loads a remote web application into the container. This is the same as browsing to the site on the device.

In order to access the device’s “features”, you need to:

  1. Pull cordova-specific javascript files into your hosted web application for the cordova project you are using
  2. Deploy and reference specific plugin modules for the functionality you want

Plugins allow Cordova to access the devices hardware, and allows you to extend the application with:

Core Plugins

  • Battery status - events on level changes, battery low or critical
  • Camera - take pictures or choose picture from gallery
  • Device - information about device such as model, version and serial numbers
  • Dialogs - pop up device-specific dialog boxes for alerts or input
  • File - allow user to upload a device file
  • Geolocation
  • Media - audio recording or playback
  • Vibration - vibrate the device

Why use Cordova?

  • Allows you to access local hardware
  • Lock user down to one app, one website

Why not to use Cordova?

  • Development can be “double-effort”
  • Local hardware like vibrate or playing a sound can be handled by javascript already (depending on browser/device)
